## 1. Purpose
The purpose of this policy is to ensure that remote working is undertaken safely from an information security perspective. Information security risks related to each specific remote working scheme must be identified, assessed, and managed.

## 2. Scope
- This policy applies to Trianz employees authorized for remote working who have remote access to Trianz IT Infrastructure and Information systems where applicable.
- It also applies to Trianz IT Infrastructure and Information systems that permit remote access for business purposes.

## 3. Policy Statements

### 3.1 Applicability
This standard applies to remote access users who have access to Trianz IT infrastructure and information assets through public networks. It is also applicable to security, system, and network engineers and administrators, as well as computer security program managers responsible for the technical aspects of preparing, operating, and securing remote access solutions and client networks.

### 3.2 Management Requirements
Before reporting managers and/or supervisors authorize work under a remote working arrangement, they must:
- Identify the type of work to be performed through the remote working arrangement.
- Only authorize remote working user access to resources necessary for carrying out the remote working arrangement safely and securely. Managers should consider if the needs can be met with less access and connectivity than provided at the Trianz office.
- Understand that work performed during an on-demand/emergency situation may not require the same access or connectivity as at the Trianz office. In many cases, VOIP, MS-Lync, telephone, and email access through a secure connection may suffice.

### 3.3 IT Administrator Requirements
Before connections are made, Trianz IT Administrators must ensure the following requirements are met:

#### 3.3.1 General Requirements
- All remote working arrangements must be approved by the IT Manager.
- Remote working users must be provided with a Trianz-owned laptop with a system image and an Internet data card to establish secure connectivity to Trianz IT Infrastructure, as needed for business purposes.

#### 3.3.2 Technical Requirements
- **Standard Build Requirements**: Laptops issued to users under telecommuting arrangements must be supported by IT personnel and adhere to standards such as:
  - Security patches
  - Anti-virus
  - Removal of administrative rights
  - Authentication
  - Auditing
  - Disabling unnecessary services
  - Password complexity
  - User privileges
  - Local security policies

- **Security Update Requirements**: All systems used for remote connectivity must be kept up to date with security updates.

### 3.3.2 Security Requirements

#### 3.3.2.1 Anti-Virus Requirements:
- All systems used to remotely connect must have Windows Defender anti-virus software installed and properly configured with the latest update.
- The anti-virus software must be configured to:
  - Scan files in real-time.
  - Scan the entire system.
  - Alert upon the discovery of a virus.

#### 3.3.2.2 Network Security Requirements:
- All systems used to remotely connect must be protected with a Trianz FortiGate UTM firewall or web gateway proxy server.
- The firewall/proxy server must be configured to block all unsolicited inbound connections.

#### 3.3.2.3 Authentication/Authorization Requirements:
- All systems used to remotely connect must require users to log in before using the system.
- Administrative rights shall be restricted.

### 3.3.3 User Requirements:

#### 3.3.3.1 General Security Requirements:
- Remote working users should connect to the internet and utilize Trianz SSL VPN to connect with Trianz IT as needed.

#### 3.3.3.2 Authentication Requirements:
- Passwords for all user accounts on the system must meet the minimum complexity requirements defined in the Trianz Password Security Policy.

#### 3.3.3.3 Attack Prevention:
- Remote working users must ensure that a combination of software and software features are installed and operating on their information assets to prevent attacks, including:
  - Anti-virus software.
  - Anti-spyware software.
- Personal firewalls must deny all types of communications that users have not specifically approved as permitted.

### 3.4 Security Maintenance and Monitoring
Remote working users must maintain the security of their information assets on an ongoing basis by:
- Periodically confirming that the operating system and primary applications are up-to-date.
- Checking the status of security software periodically to ensure it is enabled, configured properly, and up-to-date.
- Periodically checking for potential security issues on the information asset (e.g., running utilities that check the computer for potential problems).
- Investigating any cases where the information asset begins to display unusual behavior using the Security Event Viewer.

### 4. Exceptions
- There are no exceptions to this policy.
