## 1. Introduction
The purpose of the Backup and Restoration process is to ensure backup of all required information and verification of backup at periodic intervals.

## 2. Acronyms/Abbreviations
- **CM tool**: Configuration Management tool

## 3. Objective(s)
- Ensure timely availability of information assets to minimize business interruptions.
- Control and optimize the information backup process.

## 4. Scope
This procedure applies to all information backup and restoration activities performed at Trianz.

## 5. Roles and Responsibilities
- **IS Manager**: Provides approval on any user request for internal backup data and data restoration.
- **Systems Administrator**:
  - Implements backup and restoration activities as per process.
  - Updates relevant backup logs and restoration logs.
  - Maintains and stores backed-up files as specified by the process.
- **Project Manager/Delivery Manager**:
  - Provides approval on any user request for backup data and data restoration.
  - Initiates the Project Archival process.
- **Users**: Request their manager for approval to back up important data on the File Server.

## 6. Prerequisites
### 6.1 Resource
- Knowledge in Windows and other tools to back up applications.
- Knowledge of backup policies.
- Knowledge of backup devices.

### 6.2 Training
- Awareness of the Backup process.
- Familiarity with Blue Book processes.

## 7. Entry Criteria
- Scheduled backup.
- Backup/Restore is requested by a user.

## 8. Inputs
- Scheduled Backup.
- Requested Backup/Restoration: Email from user requesting the backup, with approval from the manager.

## 9. Process Description
Backup copies of information, software, and systems shall be maintained and regularly tested in accordance with the agreed topic-specific policy on backup.

### 9.1 Back Up Tasks
- Server backups will be taken regularly as per business requirements.
- Daily incremental backups will be taken from Monday to Saturday.

**9.1 Backup Process**
- Snapshots with full backups will be taken on cloud infrastructure.
- Once the backup is taken and the log is updated, an alert email will be generated.
- Refer to Appendix A for the Backup log format.

**9.2 Backup Scope**
- The following items are included in the backup scope:
  - Fileservers: users and project data
  - All Production servers
  - All Production Networking and Security Devices
  - For Cloud Services, data backup is taken over cloud storage, such as AWS Glacier/S3 or Azure Backup.

**9.3 User Requested Backup**
- Users must request backup data with the approval of the concerned Project Manager (PM).
- Based on the request, the IS Team will take the backup, and the IS Manager must approve any user request for the issuance of backup data and data restoration.
- All such cases will be maintained in the Backup log.

**9.4 Backup of User Data**
- Employees are responsible for uploading files they wish to back up to the server by placing them in their respective project directories on the file server.
- The IS Team will back up these project directories as defined above.
- Unwanted or personal files should not be placed on the server for backup; they will be deleted without notification.

**9.5 Project End Backup and Archival**
- Upon project completion or if a project is aborted, the Project Leader/PM initiates the Project Archival process. The following must be completed before archival:
  - All project-related documents and code must be checked into the Configuration Management (CM) Tool of that project.
  - All other project-related information that cannot be added to the CM Tool should be stored in a shared folder, which will also be archived along with the CM Tool database.
  - Project archivals will be included in monthly and yearly backups and will be stored/archived in Cloud/Server.
  - Configuration management audits will be performed by the CM of another project or a person familiar with the configuration management process.
  - Users can request the restoration of part or all of a backup unit.
  - Restoration must ensure data resilience and provide access to the user.
  - All restorations require email approval from the concerned manager with proper restoration details.
  - The IS Team will log the request in the backup/restore log. After restoration, the IS Team will notify the requestor.

**9.6 Backup Verification**
- The IS Team will verify daily backup logs.
- An entry indicating the success of this operation will be made in the backup log.

**9.7 Backup Restoration**
- Restoration will also be performed based on project team requests.
- For all network and security devices in the production environment, restoration tests will be performed once every six months.

**10. Output**
- The output will be stored in a repository at the cloud.

**11. Exit Criteria**
- Backup/Restoration is complete as per the plan.

**12. Measurement**
- The following metrics will be measured:
  - Number of Successful Backups vs. Number of Backups taken.
  - Number of Successful Restorations done vs. Number of Successful Backups.
